Memory Analysis of Code

The following tests were performed on an M1 Apple Macbook Air with 8GB RAM and 256GB Storage. The IDE used for these tests is Visual Studio 2022 for Mac. The memory usage was monitored using the inbuilt profiler in Visual Studio 2022 and the results attached below are the findings of the analysis. The variables in this were No of unique propositional atoms, Height of the parse tree, Time of execution in millseconds, Peak Memory Usage(KB).

The table of recordings:

Some graphs that depict the relations between the chosen variables:

This is a countplot showing the counts of the memory readings we have obtained. We can see that the range of the peak memory usage was always between 700-800KB depending on the height of the binary tree we had to store.

This is a heatmap with the correlation between each of the variables. The new correlations we can notice here are between memory in KB and the other variables. We can see the very high correlation between memory and time and also the high correlation between height of the tree and time. As the readings are fewer the correlation values are higher than they would be for more readings but this still gives an idea of what the relation between each of these variables is. These correlations obtained are not surprising as the memory will have to store more struct nodes for a longer tree and thus it will take longer to traverse this tree to evaluate the truth value.
